# 集、序列、镜头与模型时长规划

## 核心结论

AI 视频导演工作流默认使用：

```text
集 / 整条视频
  -> 序列
    -> 镜头
```

- 集是成片单位。
- 序列是 AI 视频生成单位。
- 镜头是成片剪辑单位。

视频生成提示词默认按“序列”输出。镜头不默认生成视频提示词，只写剪辑区间、画面功能和声音字幕点。

## 为什么不能按镜头直接生成

短视频剪辑镜头经常只有 1 秒、1.5 秒、2 秒或 3 秒，但主流视频模型通常有最小时长或固定时长限制。如果把镜头时长直接作为生成时长，会出现模型无法生成或浪费素材的问题。

正确做法：

```text
导演镜头时长：服务成片节奏，可以是 1.5s。
模型生成时长：服务模型限制，必须合法，例如 6s 或 8s。
剪辑使用区间：从生成序列中截取进入成片。
```

## 默认模型

默认视频模型：`Grok Imagine Video`。

默认图像模型：`GPT Image 2`。

用户本次明确指定模型时，以用户指定为准。

## 模型时长约束

以下限制变化很快，用户要求“最新”时必须联网复核。

| 模型 | 当前规划约束 | 说明 |
| --- | --- | --- |
| Grok Imagine Video | 默认按 6-15 秒规划 | xAI 官方模型页确认模型和按秒计费，但未在模型页明确列出可选时长；6-15 秒按用户当前工作流作为默认假设，实际调用前需按当前平台复核 |
| Seedance 2.0 | 5-15 秒 | Runway 官方 Seedance 2.0 页面写明 5-15 秒；其他接入平台可能显示 4-15 秒，按实际平台优先 |
| Veo 3.1 | 文生视频 4 / 6 / 8 秒；参考图转视频只支持 8 秒 | Google 官方文档明确列出 |
| Sora | 按当前 Sora 接口或产品限制复核 | 不在本 Skill 中写死 |
| Runway Gen-4 / Gen-4.5 | 按当前 Runway 页面复核 | 不在本 Skill 中写死 |
| Kling | 按当前 Kling 页面复核 | 不在本 Skill 中写死 |

## 时长合法化

生成序列前必须做时长合法化：

1. 统计序列覆盖镜头的成片总时长。
2. 选择目标模型。
3. 根据目标模型选择合法生成时长。
4. 生成时长大于成片使用时长时，规划剪辑使用区间。
5. 生成时长小于动作链所需时长时，拆成多个序列。
6. 镜头极短且不需要真实运动时，优先静态图加后期动效。

## 序列三段式模板

```text
序列 {Q01}：{序列名称}
覆盖镜头：{S01-S03}
生成模型：{Grok Imagine Video}
生成方式：{文生视频/图生视频/首帧视频/首尾帧视频/多参考图视频}
生成时长：{合法时长}
成片使用：{预计使用时长}
参考资产：{定妆照/场景图/道具图/风格帧/首帧/尾帧}

【基础设定】
{时间、地点、主体、角色、道具、参考资产用途、声音边界}

【氛围与画质】
{风格核心、真实感、摄影质感、色彩影调、光线、限制词}

【画面内容】
{按生成时长拆分连续动作链，标注可剪辑节点、景别变化、构图变化、运镜和声音细节}
```

## 镜头剪辑表示例

```text
S01 / 1.5s / Q01 / 0.5-2.0s / 建立危险 / 山路远景，远处山洪出现 / 远景 / 引导线 / 轻微推近 / 雷声和水流声 / 无字幕 / 开头快速切入
S02 / 1.0s / Q01 / 2.6-3.6s / 强化冲击 / 泥水裹挟树枝冲上路面 / 特写 / 对角线 / 手持跟随 / 水流轰鸣 / 无字幕 / 可加低频冲击音效
S03 / 1.5s / Q01 / 4.3-5.8s / 危机升级 / 水流横扫公路冲向镜头 / 广角近景 / 居中压迫 / 轻微后退 / 水花拍击镜头 / 无字幕 / 接司机反应镜头
```

## 常用处理策略

- 短镜头合并：同一场景、同一角色、同一动作链的多个短镜头合并为一个序列生成。
- 延展后裁切：生成 6 秒或 8 秒连续动作，从中截取 1-3 秒进入成片。
- 静态图加动效：反应特写、产品细节、字幕信息镜头可用定妆照或道具图做推拉摇移。
- 变速：动作镜头可生成较长片段后加速剪辑；口型对白不建议变速。
- 拆序列：动作链太长、场景变化、角色状态变化或参考资产变化时，拆成多个序列。
