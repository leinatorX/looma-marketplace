# 剧本、序列与镜头设计

## 剧本结构

短视频剧本应优先服务留存，而不是完整电影式铺垫。

### 15 秒结构

```text
0-3 秒：钩子，直接给冲突、结果、反差或强视觉。
3-10 秒：递进，展示过程、证据、变化或危险升级。
10-15 秒：高潮或结论，给反转、答案、行动或循环点。
```

### 30 秒结构

```text
0-3 秒：钩子。
3-8 秒：建立人物、问题或场景。
8-20 秒：冲突升级或信息递进。
20-27 秒：解决、反转或视觉高潮。
27-30 秒：记忆点、行动号召或循环回看。
```

## 序列设计原则

- 序列是 AI 视频生成单位，不是传统剪辑镜头。
- 一个序列应尽量保持同一场景、同一角色组、同一动作链、同一光线和同一参考资产组。
- 一个序列可以包含多个镜头，镜头从序列中裁切。
- 每个序列必须满足目标模型的合法生成时长。
- 序列提示词使用三段式：基础设定、氛围与画质、画面内容。
- 镜头层只记录剪辑信息，除非用户明确要求或镜头满足模型合法时长并需要独立生成。

## 镜头设计原则

- 一个镜头只表达一个主要动作或信息。
- 开头镜头必须有明确主体和明确变化，避免空镜铺垫太长。
- AI 不稳定时，把复杂动作拆成多个 2-4 秒短镜头。
- 人物口型、手部、群体动作、复杂物理交互容易失败，需要降低复杂度。
- 角色连续出现时，每个镜头都重复稳定特征：发型、服装、脸型、材质、关键道具。
- 角色、产品、品牌、道具或主场景需要连续出现时，优先把相关镜头放入同一图生视频序列，不要只靠文生视频维持一致。
- 动作起点和终点都重要时，优先规划首尾帧。
- 需要剪辑的镜头不要把所有信息塞满，给转场、字幕和音效留空间。

## 分镜类型

### 剧情短片

使用“冲突递进”：

```text
异常出现 -> 主角反应 -> 危机升级 -> 选择动作 -> 结果或反转
```

### 产品展示

使用“问题-产品-证据-结果”：

```text
痛点画面 -> 产品出现 -> 功能演示 -> 使用前后对比 -> 购买/了解理由
```

### 知识科普

使用“误区-解释-示范-结论”：

```text
常见误解 -> 关键原理 -> 可视化演示 -> 一句话总结
```

### 视觉奇观

使用“尺度升级”：

```text
局部细节 -> 主体变化 -> 大场面展开 -> 震撼收束
```

## 序列提示词模板

```text
序列 {编号}，{合法生成时长} 秒，{画幅}。
覆盖镜头：{S01-S03}。
剧情功能：{这个序列要推动什么剧情或情绪}。
生成方式：{文生视频/图生视频/首帧视频/首尾帧视频/多参考图视频}。
参考资产：{角色定妆照/场景图/道具图/风格帧/首帧/尾帧}。

【基础设定】
{时间、地点、主体、外观、道具、参考资产用途、声音边界}

【氛围与画质】
{光线、色彩、质感、影像风格、限制词}

【画面内容】
{按时间段描述连续动作链、可剪辑节点、景别变化、构图变化、运镜和声音细节}

负面约束：避免 {不想要的错误}。
```
