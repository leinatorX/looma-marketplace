# GPT-Image2 提示词规范

本文件面向 Looma `image-generator`（展示名 ImageGenerator）生成图片时的实际执行。

## 核心结论

不要把提示词写成关键词堆叠。把用户需求改写成一份明确的视觉设计任务书，按以下顺序表达：

如无特殊要求，提交给接口的最终提示词一律使用简体中文编写。只有用户明确要求英文提示词、目标图片必须包含英文文案、或专有英文术语不可翻译时，才保留英文。

1. 用途 / 平台
2. 主题 / 目标
3. 画面主体
4. 场景环境
5. 构图方式
6. 视觉风格
7. 必须出现的文字
8. 必须保留或禁止改变的内容
9. 负面约束
10. 输出比例 / 尺寸 / 质量要求

推荐工作句式：

```text
先说明图片用来做什么；
再说明画面中有什么；
再说明文字写什么；
再说明风格和构图；
最后说明不要什么。
```

## 提示词基础模板

```text
生成一张【比例 / 接口尺寸】的高质量图片，用于【使用场景 / 平台】。

【主题】
图片主题是：【具体主题】。

【目标】
这张图的目标是：【吸引点击 / 建立认知 / 科普解释 / 产品展示 / 形成收藏 / 引发评论 / 转发传播】。

【画面主体】
画面核心主体是：【人物 / 产品 / 场景 / 设备 / 图标 / 数据关系 / 知识结构】。
主体应位于：【居中 / 左侧 / 右侧 / 前景 / 中景 / 背景】。
主体特征：【姿态、表情、材质、颜色、动作、状态】。

【场景环境】
背景环境是：【室内 / 户外 / 工厂 / 办公室 / 城市 / 实验室 / 消防现场 / 纯色背景】。
环境氛围：【专业、干净、真实、科技感、紧张、温暖、秩序感、未来感】。

【构图】
采用：【中心主视觉 + 信息卡 / 左右对比 / 上中下结构 / 时间轴 / 流程图 / 知识图谱 / 产品广告构图】。
画面层次清楚，主次分明，留白合理，适合移动端阅读。

【视觉风格】
整体风格为：【写实摄影 / 3D卡通 / 扁平插画 / 现代信息图 / 电影感 / 科技蓝 / 小红书图文风 / 企业宣传风】。
要求画面高级、干净、有视觉吸引力，不要像传统PPT。

【文字要求】
必须出现以下中文文字，逐字准确：
主标题：“……”
副标题：“……”
信息点：
1. “……”
2. “……”
3. “……”

所有中文必须清晰、准确、无乱码、无错别字、无伪中文。
不要添加提示词之外的多余文字。

【限制】
不要水印。
不要Logo，除非明确要求。
不要英文乱码。
不要模糊文字。
不要过度拥挤。
不要把所有信息一次性堆满。
不要出现与主题无关的元素。
```

## 中文文字入图规则

- 文字必须使用简体中文。
- 只允许出现用户指定的文字，不要自动添加额外标题、英文、编号或装饰性文字。
- 主标题最大，副标题次之，正文信息点清楚分组。
- 文字和背景要有足够对比度，不要让文字压在复杂背景上。
- 文字较多时拆成信息卡片，每张卡片只放一组信息。
- 每张图只讲一个核心问题，不要在一张图里塞完整文章。

## 平台比例与接口安全尺寸

原指南中的平台尺寸是发布展示基准；`gpt-image-2` 接口还要求宽高为 16 的倍数，且总像素数至少 `655360`。因此执行时优先使用下表的接口安全尺寸，发布前再按平台需要裁切或缩放。

| 场景 | 展示基准 | 接口安全尺寸 | 说明 |
|---|---:|---:|---|
| 公众号头条封面 | 900x383 / 2.35:1 | 1536x656 | 接近 2.35:1，生成后可裁切到 900x383 |
| 公众号次条封面 | 1:1 | 1024x1024 | 方图最稳定 |
| 公众号正文配图 | 900 或 1080 宽 | 1536x1024 / 1024x1536 | 按横图或竖图内容选择 |
| 小红书图文 | 1080x1440 / 3:4 | 1536x2048 | 适合知识卡、封面图 |
| 视频号竖版封面 | 1080x1260 / 6:7 | 1088x1264 | 接近 6:7，避免误用 9:16 |
| 视频号横版封面 | 16:9 | 2048x1152 | 横版封面、横屏内容 |
| 抖音 / 快手竖屏封面 | 1080x1920 / 9:16 | 1088x1936 | 核心主体和标题放中间安全区 |
| 横屏视频 / PPT / B站 | 16:9 | 2048x1152 | 比 1920x1080 更符合接口倍数要求 |
| 方形头像 / Logo | 1:1 | 1024x1024 | Logo 类不要添加未授权品牌标识 |

不要直接把 `900x383`、`1080x1440`、`1920x1080` 作为接口 `size`，这些尺寸不满足接口约束。

## 图像编辑与多图参考

局部编辑时，明确“只改什么”和“必须保留什么”：

```text
请基于上传图片进行编辑。
只修改：【具体区域 / 对象】。
把它改成：【目标效果】。
保留原图的人物脸部、姿态、构图、镜头角度、光影方向、背景主体和画面比例。
不要改变人物身份，不要新增无关元素，不要改变画面整体风格。
修改后要自然、真实、融合原图。
```

多图参考时，给每张图分配角色：

```text
请参考上传的多张图片生成新图。
图1：作为【主体 / 产品 / 人物】参考。
图2：作为【风格 / 配色 / 构图】参考。
图3：作为【场景 / 背景 / 氛围】参考。
生成一张新的【接口尺寸】图片：
- 保留图1的主体特征
- 使用图2的视觉风格
- 放入图3类似的场景氛围
- 不要直接拼贴
- 不要照搬原图文字
- 生成一张自然统一的新画面
```

## 安全科普 / 应急类规则

- 画面要专业、克制、准确。
- 可以表现风险，但不要过度渲染灾难。
- 不要血腥、尸体、严重受伤、恐怖画面。
- 不要夸张爆炸和大面积伤亡场景。
- 不要误导公众进行危险操作。
- 涉及应急处置时，优先强调远离危险、报警、听从专业人员指挥。
- 涉及法规、标准、数字、标识时，生成后必须人工复核。

## 出图前自检

1. 是否明确用途和平台？
2. 是否明确比例和接口合法尺寸？
3. 是否明确主题和目标？
4. 是否指定主标题和副标题？
5. 是否写清楚必须出现的文字？
6. 是否写清楚画面主体和构图？
7. 是否写清楚风格？
8. 是否写清楚不要什么？
9. 是否避免一张图塞太多信息？
10. 是否避免平台比例错误？

## 出图后自检

1. 主题是否一眼能看懂？
2. 主标题是否清晰？
3. 中文是否有乱码、错别字、伪中文？
4. 是否出现提示词之外的多余文字？
5. 主体是否突出？
6. 构图是否拥挤？
7. 信息是否提前泄露太多？
8. 是否像 PPT 而不是传播图？
9. 视觉风格是否统一？
10. 是否存在安全、平台审核或事实表达风险？
11. 人物、设备、车辆、标识是否结构合理？
12. 是否适合手机端缩略图观看？
13. 是否符合目标平台比例？
14. 公众号封面是否接近 2.35:1，而不是 3:4？
15. 视频号竖版是否接近 6:7，而不是误用 9:16？
16. 是否已把图片真正发送到当前会话，而不只是给出路径？
